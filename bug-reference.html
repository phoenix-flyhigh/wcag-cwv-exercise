Operable — WCAG Principle 2

2.1 Keyboard Accessible · 2.4 Navigable
# Bug WCAG Severity Location Fixed?
15 Global outline: none on *:focus — every focusable element on the page has its focus ring stripped. Keyboard users have zero indication of where focus is.
Remove *:focus { outline: none }. If custom styles needed, use :focus-visible with a visible custom outline.
 2.4.7 AA CRITICAL CSS: *:focus
16 Positive tabindex values on nav links — tabindex="1" through tabindex="5" are set out of DOM order (Women=1, Men=2, Kids=3, Sale=4, Contact=5). Tab order jumps illogically through the page.
Remove all positive tabindex values. Let natural DOM order determine tab order.
 2.4.3 A CRITICAL .nav-links a
17 Clickable product cards are not keyboard accessible — .product-card uses onclick on a <div>. No tabindex, no role. Keyboard users cannot activate them.
Add tabindex="0" + role="button" + keydown handler (Enter/Space). Or restructure so the product name is an <a>.
 2.1.1 A CRITICAL .product-card
18 Modal: focus not moved when opened — clicking a product card opens a modal, but focus stays behind the modal on the card. Screen reader users don't know the modal exists.
On open: modalEl.focus() or move focus to the first interactive element inside. Set document.body.inert = true or trap focus inside modal.
 2.1.2 A 2.4.3 A CRITICAL openModal() in JS
19 Modal: Escape key not handled — pressing Escape does not close the modal. Required behavior for all modal dialogs.
Add document.addEventListener('keydown', e => { if (e.key === 'Escape') closeModal(); })
2.1.1 A MAJOR JS — modal
20 Tooltip only on hover, not on keyboard focus — the info tooltip only appears on CSS :hover. Keyboard users tabbing to the button never see the tooltip.
Show tooltip on :focus-visible too. Or use aria-describedby linking to a role="tooltip" span and show/hide it on focus events.
 2.1.1 A MAJOR .tooltip-wrap
21 No skip navigation link — pages with repeated navigation must have a mechanism to skip to main content. Keyboard users must tab through every nav item on every page.
Add <a href="#main-content" class="skip-link">Skip to main content</a> as the very first element. Style it to appear only on focus.
 2.4.1 A MAJOR Top of <body>
22 Custom checkbox is not keyboard accessible — the "I agree" checkbox is a <div> with an onclick. Cannot be toggled with keyboard. No checked state announced.
Replace with a real <input type="checkbox"> with a <label>, or add role="checkbox", tabindex="0", aria-checked, and keyboard handlers.
 2.1.1 A 4.1.2 A CRITICAL #terms-check

# Bug WCAG Severity Location Fixed?
23 Missing lang attribute on <html> — without a language declaration, screen readers may use the wrong voice/pronunciation engine. Critical for Indian-language content.
Add <html lang="en"> (or "en-IN" for Indian English).

Add aria-live="polite" (or "assertive") to the error container, or use aria-describedby linking each input to its error div + aria-invalid="true" on the input.

25 Email input uses type="text" — misses browser-native email validation, autocomplete hints, and mobile keyboard optimization (no @ key on standard keyboard layout).

Core Web Vitals

CLS · LCP · INP — Measured via Lighthouse / PageSpeed Insights / CrUX
# Bug Metric Severity Location Fixed?
CWV-1 Hero image has no width/height attributes — without explicit dimensions, the browser doesn't reserve space. When the image loads, content below jumps down.
Add width="1400" height="500" attributes on the img tag, or set aspect-ratio: 1400/500 in CSS.
 CLS CRITICAL .hero-image
CWV-2 Product images injected via JS with 800ms delay — the product grid is empty on initial render. After 800ms, 8 images load without dimensions, causing a massive CLS event as the grid reflows.
Server-render or inline the product HTML. If JS-injected, set explicit width/height on images and reserve grid height with a skeleton loader.
 CLS LCP CRITICAL setTimeout in JS (800ms)
CWV-3 Flash banner injected at 1200ms — a "Flash Deal" banner is injected via JS after 1.2 seconds, pushing all content below it down. Classic late-injection CLS.
Pre-allocate space with min-height: 40px on the banner container, or server-render the content. Never inject banners that shift layout after paint.
 CLS CRITICAL #flash-banner setTimeout
CWV-4 Announcement div animates height from 0 → 48px — the CSS @keyframes popIn animates height property. Height animations trigger layout, causing CLS on every page load.
Animate transform: scaleY() or opacity instead of height. Both are compositor-friendly and don't cause layout shifts.
 CLS MAJOR .announcement / @keyframes popIn
CWV-5 Carousel images have no explicit dimensions — carousel is overflow: hidden with no set height. As images load, the carousel height shifts.
Set aspect-ratio: 4/1 on .carousel, or give images width/height attributes.
 CLS MAJOR .carousel-track img
CWV-6 Hero image is the LCP element — not preloaded — the largest contentful paint is the hero image, but there's no <link rel="preload">. Browser discovers it only when parsing the <img> tag, delaying LCP.
Add <link rel="preload" as="image" href="..." fetchpriority="high"> in the <head>. Also add fetchpriority="high" attribute directly on the img tag.
 LCP CRITICAL <head> / .hero-image
CWV-7 Google Fonts loaded synchronously without preconnect — the stylesheet link blocks rendering while the browser establishes a connection to fonts.googleapis.com. No preconnect hint provided.
Add <link rel="preconnect" href="https://fonts.googleapis.com"> and <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> before the font CSS link. Also consider font-display: swap.
 LCP MAJOR <head> fonts link
CWV-8 Scroll handler calls offsetTop on every scroll event — element.offsetTop forces a layout recalculation (layout thrashing). Called on every scroll event without debouncing or requestAnimationFrame, causing jank and degrading INP.
Use IntersectionObserver instead. Or at minimum, debounce the scroll handler and use requestAnimationFrame to batch reads.
 INP MAJOR window scroll listener in JS
